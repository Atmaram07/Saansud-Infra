# Saansud Infra Website

## Overview

This repository contains a static multi-page marketing website for **Saansud Infra** with a small PHP mail layer for the contact and quote forms.

At a high level, the project is:

- A brochure-style company website
- Built with plain HTML, CSS, JavaScript, jQuery, and Bootstrap
- Organized as individual pages instead of a framework or CMS
- Enhanced with sliders, animations, video backgrounds, and AJAX form submission
- Split into two main content tracks:
  - Core Saansud real-estate / brand pages
  - Construction and interior-services landing pages

Current repo snapshot:

- `15` HTML files in the root
- `5` CSS files
- `10` JavaScript files
- `2` PHP form handlers
- `239` asset files under `assets/`

## Site Architecture

### High-level structure

The website is not generated by a build system. Each page is a standalone HTML file that repeats the same basic pattern:

1. Shared CSS files are linked from `css/`
2. jQuery is loaded first
3. Page-specific markup is written inline in the HTML file
4. The footer is injected at runtime from `footer.html` using a custom `<include>` tag plus `fetch()`
5. Shared JS libraries and `js/main.js` are loaded at the bottom
6. Some pages also load `js/email.js` for form submission

### Shared layout pattern

Most pages follow this structure:

- Header with navbar
- Main content
- `<include src="footer.html"></include>`
- Inline script that fetches and injects `footer.html`
- Shared JS library includes

### Important implication

Because the footer is loaded through `fetch()`, the site should be served over **HTTP/HTTPS**, not opened directly via `file://`. If opened as local files in a browser, footer injection may fail due to browser security rules.

## Shared Runtime Patterns

### Footer injection

Every main HTML page uses:

```html
<include src="footer.html"></include>
```

Then an inline script:

```js
document.querySelectorAll("include").forEach(async (el) => {
  let file = el.getAttribute("src");
  let resp = await fetch(file);
  let html = await resp.text();
  el.outerHTML = html;
});
```

This is a lightweight include system for static pages.

### Navbar pattern

The header is duplicated across pages rather than shared through a server-side template. Active nav classes are manually set per page.

### Slider pattern

Swiper containers are created directly in page markup and activated centrally in `js/main.js`.

## Local Development

### How to run

Because there is no build step, development is simple:

1. Serve the folder with any local web server
2. Open `index.html` through that server
3. Ensure PHP is available if you want to test forms

Example options:

- XAMPP / WAMP
- PHP built-in server
- VS Code Live Server for frontend-only testing

### Form testing

Frontend-only:

- You can verify layout and AJAX calls
- The request will fail unless PHP is available

Full form testing:

- Serve the repo through PHP
- Configure working mail transport on the server
